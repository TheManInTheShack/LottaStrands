# Session 2026-03-30

First session with this project. Covered a large amount of ground — re-examined the repo, established architecture, and built the majority of the working system.

## Done

**Architecture & naming**
- Established property graph model (JSON now, Neo4j later); nodes + edges with properties
- Defined PLOT acronym: Parse → Lexicon → Ontology → Taxonomy
- Named the product **Plotz** / plotzed.com; purchased domain
- Clarified: LottaStrands = repo/corpus name; Plotz = product/site name
- Corpus = implicit (the instance IS the corpus); users add Volumes
- Removed redundant "CORPUS" label from UI

**Backend**
- `engine/graph/model.py` — Node, Edge, Graph; JSON save/load
- `engine/ingest/flat_ingest.py` — ingests raw text as one scene, paragraphs by blank lines; now appends to existing corpus instead of replacing
- `engine/curate/operations.py` — merge_nodes, split_node, rename_node, apply_operations
- `engine/api/` — FastAPI with state module; routes for corpus, paragraphs, volumes, curate
- `POST /volumes` — creates/appends volume; resets curation.json (bakes in curated state as new baseline)
- `POST /volumes/delete` — BFS removal of volume subgraph; shared nodes untouched
- `POST /curate/insert_marker` — splits scene at paragraph boundary by paragraph ID
- Fixed curation double-replay bug: state.load() was re-running ops on already-curated graph

**Godot UI**
- CorpusMenu: 30/40/30 margins, stacked layout, volume list with VolumeListItem (marquee scroll on hover, delete button)
- New Volume form: inline panel, waits for API response before closing, shows errors
- Delete confirmation: ConfirmationDialog with per-volume warning; cancel resets hover state
- Main: header + full-width CurationView (replaced old SceneDetail + SceneList split)
- CurationView: paragraph cards (ParagraphCard), insert strips (InsertStrip, ÷ icon), scene boundary markers (HierarchyMarker, removable)
- API.gd / AppState.gd: volume_created signal; all new endpoints wired

**Documentation**
- README updated with Plotz name and etymology
- hierarchy.json stripped to corpus-level config (source block was per-volume, not corpus)
- This Obsidian vault created; CLAUDE.md files at global and repo level

## Decided
- Flat ingest (blank-line paragraphs, single "FULL TEXT" scene) replaces old auto-parser — human curation is the right path; automated parse is too brittle
- curation.json is reset on every volume add — avoids double-replay; past curation is baked into graph.json as the new baseline
- Lexicon / Ontology / Taxonomy will live as non-Volume-parented nodes in graph.json (naturally shared across volumes)
- No right-click menus; no hover-only interactions — phone/tablet is primary device
- InsertStrip tap pattern for inserting hierarchy markers (not context menu)
